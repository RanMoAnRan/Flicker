# 音乐库锁屏续播设计

## 背景

当前音乐库使用原生 `<audio>` 元素播放远程音频，基础能力已经具备，但缺少系统级媒体会话、播放状态持久化和前后台切换恢复逻辑，导致移动端锁屏时体验不稳定，自动下一首也容易受浏览器回收或状态丢失影响。

## 目标

- 移动端锁屏时尽量保持音频继续播放。
- 当前歌曲结束后，在浏览器允许后台媒体事件继续运行时自动切到下一首。
- 锁屏媒体面板展示歌曲标题、歌手、封面，并支持系统上一首、下一首、播放、暂停。
- 页面回到前台后，尽量恢复上一次的播放上下文和队列状态。

## 方案

1. 保留原生 `<audio>` 作为底层播放器，避免改为兼容性更差的 Web Audio 驱动。
2. 通过 `Media Session API` 同步媒体元数据、播放状态和系统媒体按键回调。
3. 将当前队列、当前歌曲、播放位置、播放地址和预期播放状态写入本地存储。
4. 页面重新进入前台时，若上一次状态应为播放中且音频已暂停，则尝试温和恢复。
5. 保留现有 `ended -> stepQueue(1)` 逻辑，并增加切歌过程中的并发保护，避免重复自动切歌。

## 非目标

- 不承诺所有 iPhone 和安卓浏览器在所有省电策略下都 100% 后台不断播。
- 不引入 Service Worker 音频代理、PWA 安装流程或原生 App 能力。
