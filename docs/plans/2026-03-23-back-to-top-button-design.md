# 首页与音乐页回到顶部按钮设计

## 背景

首页与音乐库页都是长滚动页面，滚动超过一屏后回到顶部成本较高。当前站点没有统一的“回到顶部”能力，需要补一个轻量、可复用的公共交互。

## 目标

- 在视频库首页和音乐库页提供统一的回到顶部按钮
- 页面滚动超过一屏后再显示，避免首屏干扰
- 点击后平滑滚动回到顶部
- 音乐库页需要避开底部固定播放器，不遮挡主要操作

## 方案

采用公共脚本注入方案：

- 在 `js/app.js` 中新增通用 `BackToTopManager`
- 仅在 `home-page` 与 `music-page` 两类页面启用
- 由脚本在 `body` 末尾注入按钮，避免分别改多个页面模板
- 样式统一放在 `css/style.css`

## 交互定义

- 触发阈值：页面滚动距离超过当前视口高度
- 显示方式：右下角淡入出现
- 点击行为：平滑滚动到页面顶部
- 动画策略：仅使用轻微位移与透明度过渡

## 页面适配

### 首页

- 使用默认右下角悬浮位置
- 不干扰内容区与分页区域

### 音乐页

- 保持相同按钮样式
- 底部位置上移，避开固定播放器
- 在窄屏断点继续保留额外底部安全距离

## 风险与验证

- 风险：音乐页按钮可能与播放器或抽屉入口冲突
- 对策：为 `music-page` 单独设置更高的底部偏移
- 验证：
  - 首页滚过一屏后出现按钮，点击可回顶
  - 音乐页滚过一屏后出现按钮，且不压住播放器
  - 返回顶部动画在支持环境下为平滑滚动
