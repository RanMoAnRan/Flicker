# 浮光音乐库接入设计

## 目标

在现有影视站基础上新增“音乐库”入口，并通过本地 Node 代理接入 `https://musicfreepluginshub.2020818.xyz/plugins.json` 中的白名单音乐插件，实现第一版：

- 查看音乐插件目录
- 按关键词搜索歌曲
- 点击结果后在线播放

## 约束

- 不在浏览器直接执行远程插件代码
- 第一版仅启用白名单插件，优先接入 `W音乐`
- 现有影视首页、详情页、播放页行为不回归

## 方案

### 服务端

- 代理层新增音乐接口：
  - `/api/music/plugins`
  - `/api/music/search`
  - `/api/music/media`
- 代理层拉取并缓存 `plugins.json`
- 仅允许白名单插件进入可用列表
- 使用 `vm` 在 Node 侧加载插件，并注入受控 `env`
- 使用本地 shim 提供 `dayjs`、`crypto-js` 等插件运行必需能力

### 前端

- 首页头部新增“音乐库”按钮
- 新增 `music.html`：
  - 音乐搜索框
  - 插件来源列表
  - 搜索结果卡片
  - 底部播放器

## 第一版范围

- 支持 `W音乐` 插件搜索
- 支持获取歌曲播放地址并在线播放
- 支持展示封面、歌名、歌手、音质
- 暂不处理：
  - 登录态注入
  - 收藏/歌单
  - 歌词滚动
  - 音频代理和 Range 转发

## 风险

- 第三方插件可能随时变更
- 远程接口可能限流或失效
- 部分音源可能返回可搜索但不可播的条目

## 取舍

第一版优先把“入口、搜索、播放”主链路打通，后续再增加更多插件和更强的稳定性措施。
