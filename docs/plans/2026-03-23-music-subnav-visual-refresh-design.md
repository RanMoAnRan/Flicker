# 音乐库插件二级导航统一设计

## 背景

当前音乐库已将“推荐音乐插件 / 其他音乐插件 / 非音乐插件”上移到头部下方，但视觉和结构仍不完全统一：

- 桌面端顶部区域更像一张独立内容卡片，而不是页面级二级导航
- 移动端继续保留内容区 inline tab，与桌面端形成两套导航结构
- 同一份插件分类被重复渲染，层级和维护成本都偏高

## 目标

- 让该区域更像网站频道栏，而不是内容模块
- 降低装饰性，提升信息架构清晰度
- 保持当前交互和信息不变，同时统一桌面端与移动端的导航结构
- 移动端改为与桌面端共用头部二级导航，不再在内容区重复渲染 tab

## 方案

采用“统一频道栏”方案：

- 保留头部下方的 dock 位置，并扩展到移动端
- 去掉外层卡片的明显背景、阴影和大圆角
- 将 tab 组改为更轻的横向导航条
- 默认态以文字导航为主，数量信息降级为辅助信息
- 选中态改为细下划线和更高文字对比，不再使用整块高亮按钮
- 内容区仅保留当前分类下的插件列表，不再重复放置 tab

## 具体调整

### 外层容器

- dock 区域使用与 header 接近的轻透明背景
- 通过细边线分隔头部与内容区
- 不再单独模拟一张浮起卡片

### tab 导航

- 桌面端与移动端都使用头部 dock 内的 tab
- 默认态无大面积底色、无按钮感阴影
- 选中态使用品牌色下划线和更高字重
- hover 仅做轻微文字色变化
- 移动端支持横向滑动浏览 tab，保证窄屏下仍能完整展示

### 信息权重

- 主标题保持为 tab 名称
- 数量 `xx 个` 保留，但字号更小、颜色更弱
- 避免数量信息主导视觉

## 风险与验证

- 风险：过于极简后可能导致选中态不够明显
- 对策：通过底部强调线、文字对比和字重提升确保可识别性
- 验证：
  - 桌面端确认更像二级导航而非卡片
  - 移动端确认与桌面端使用同一套头部二级导航
  - 内容区不再重复出现插件分类 tab
  - tab 切换逻辑保持不变
