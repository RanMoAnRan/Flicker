# 音乐库桌面端二级导航视觉调整设计

## 背景

当前音乐库桌面端已将“推荐音乐插件 / 其他音乐插件 / 非音乐插件”上移到头部下方，但视觉呈现仍更像一张独立内容卡片，而不是页面级二级导航。主要问题：

- 外层容器过厚，卡片感明显
- 阴影、渐变、圆角较重，视觉语言偏花
- 与顶部搜索区同时争夺注意力，层级不够清晰

## 目标

- 让该区域更像网站频道栏，而不是内容模块
- 降低装饰性，提升信息架构清晰度
- 保持当前交互和信息不变，仅调整桌面端视觉表达
- 移动端继续保留原有内容区 tab 形式，不强行统一

## 方案

采用“极简频道栏”方案：

- 保留头部下方的桌面端 dock 位置
- 去掉外层卡片的明显背景、阴影和大圆角
- 将 tab 组改为更轻的横向导航条
- 默认态以文字导航为主，数量信息降级为辅助信息
- 选中态改为细下划线和更高文字对比，不再使用整块高亮按钮

## 具体调整

### 外层容器

- dock 区域使用与 header 接近的轻透明背景
- 通过细边线分隔头部与内容区
- 不再单独模拟一张浮起卡片

### tab 导航

- 桌面端 dock 内的 tab 改为横向排列
- 默认态无大面积底色、无按钮感阴影
- 选中态使用品牌色下划线和更高字重
- hover 仅做轻微文字色变化

### 信息权重

- 主标题保持为 tab 名称
- 数量 `xx 个` 保留，但字号更小、颜色更弱
- 避免数量信息主导视觉

## 风险与验证

- 风险：过于极简后可能导致选中态不够明显
- 对策：通过底部强调线、文字对比和字重提升确保可识别性
- 验证：
  - 桌面端确认更像二级导航而非卡片
  - 移动端维持现有 inline tab，不受影响
  - tab 切换逻辑保持不变
