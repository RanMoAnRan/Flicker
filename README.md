# 浮光

一个基于静态页面和本地 Node.js 代理服务的影视资源聚合站点原型，当前已经扩展出首版“音乐库”入口。

项目当前不依赖前端构建工具，页面直接由浏览器加载 HTML、CSS、JavaScript，本地服务负责两件事：

- 提供静态文件访问
- 通过 `/api/proxy` 转发外部 CMS 接口请求，规避浏览器跨域限制

这份文档面向开发者，重点说明项目结构、运行方式、核心模块和维护要点。

## 项目特性

- 首页浏览当前片源的分类内容、推荐内容和分页列表
- 支持关键词搜索，并可聚合多个已配置片源的搜索结果
- 支持从首页进入“音乐库”，按关键词搜索白名单音乐插件并在线播放
- 音乐库支持加载歌词，并随播放进度高亮当前歌词
- 音乐库支持上一首、下一首和基于当前搜索结果的播放队列
- 支持 `vod` 影视源与 `art` 资讯源两类接口
- 详情页可根据 `source_id` 回到正确片源加载详情
- 播放页支持多线路切换，并优先识别 M3U8 播放源
- 设置页支持片源新增、编辑、删除、切换和健康检查
- 使用 `localStorage` 保存片源配置、当前片源、主题和海报缓存
- 本地代理服务支持常见网络错误映射，便于前端展示更清晰的失败原因

## 技术栈

- 前端：原生 HTML / CSS / JavaScript
- 服务端：Node.js 原生 `http` 模块
- 请求代理：`curl` + Node 子进程
- 依赖：
  - `express`
  - `cors`
  - `node-fetch`

说明：

- 当前 `proxy-server.js` 实际使用的是 Node 原生 `http` 实现，而不是 Express。
- `package.json` 中保留了 `express`、`cors`、`node-fetch` 依赖，但当前主流程未直接使用它们。

npm install

## 部署到 Render

如果你没有自己的服务器，当前项目最省事的上线方式就是直接部署到 Render。这个项目本身就是前后端一体的单个 Node 服务，Render 会帮你从 GitHub 拉代码、安装依赖并运行 `npm start`。

项目里已经补好了 [render.yaml](/Users/ranmoanran/Jing/python-space/Flicker/render.yaml)，可以直接按下面步骤操作。

### 部署步骤

1. 把当前仓库推送到 GitHub。
2. 打开 Render 并登录：<https://render.com/>
3. 在 Dashboard 中选择 `New +` -> `Blueprint`。
4. 连接你的 GitHub 仓库，并选择当前项目。
5. Render 会自动识别仓库中的 `render.yaml`。
6. 确认服务配置后点击创建，等待首次构建完成。
7. 部署成功后，Render 会分配一个 `onrender.com` 域名，直接访问即可。

### 当前部署配置

- 服务类型：Node Web Service
- 构建命令：`npm install`
- 启动命令：`npm start`
- 健康检查：`/`
- Node 版本：`20`

### 为什么这个方案适合当前项目

- 不需要额外服务器。
- 不需要拆分前端和后端。
- 页面文件和 `/api/proxy`、`/api/music/*` 接口都由同一个服务提供。
- Render 原生运行时包含 `curl`，能满足当前代理和音乐插件链路。

### 部署后的验证建议

部署成功后，建议至少检查这几个地址：

- 首页：`https://你的域名/`
- 音乐库：`https://你的域名/music.html`
- 插件目录接口：`https://你的域名/api/music/plugins`
- 通用代理接口示例：`https://你的域名/api/proxy?url=https://www.baidu.com`

### 常见问题

#### 1. 首次打开比较慢

如果你使用的是免费实例，可能会出现冷启动，第一次访问会慢一些，属于正常现象。

#### 2. 音乐插件偶发失效

音乐库依赖第三方插件和上游站点，某些搜索、播放或歌词异常通常不是 Render 本身的问题，而是远程插件或上游源变化导致。

#### 3. 构建成功但接口异常

先检查：

- Render 日志里服务是否正常启动
- 访问 [music.html](/Users/ranmoanran/Jing/python-space/Flicker/music.html) 时浏览器控制台是否有报错
- `/api/music/plugins` 是否能正常返回 JSON

如果后续你想完全固定运行环境，或者需要额外系统依赖，再把当前项目切到 Docker 部署会更稳。

## 页面说明

### 首页 `index.html`

首页负责片库浏览、搜索、筛选和分页展示。

主要能力：

- 读取当前片源并加载分类
- 展示推荐区、瀑布流列表和分页
- 支持关键词搜索
- 支持聚合搜索多个片源
- 支持按内容类型和状态筛选
- 支持从 URL 中恢复 `page`、`category`、`wd` 等状态

### 详情页 `detail.html`

详情页根据 URL 参数中的影片 ID 加载详情数据，并优先使用 `source_id` 定位片源。

主要能力：

- 展示影片或资讯详情
- 展示海报、简介、元信息、标签
- 为影视内容提供播放入口
- 为资讯内容提供阅读展示

### 播放页 `player.html`

播放页负责解析播放线路和剧集列表。

主要能力：

- 读取 `id`、`ep`、`source`、`source_id`
- 解析 `vod_play_from` 和 `vod_play_url`
- 按线路切换播放源
- 优先识别 M3U8 线路
- 对 `art` 资讯源显示不可播放提示

### 音乐库页 `music.html`

音乐库页负责直连 MusicFree 远程插件目录，并在站内提供搜索与在线播放。

主要能力：

- 从首页一键进入音乐库
- 读取 `/api/music/plugins` 获取远程插件目录及推荐插件状态
- 通过 `/api/music/search` 执行关键词搜索
- 通过 `/api/music/media` 获取歌曲播放地址
- 通过 `/api/music/lyric` 获取歌曲歌词
- 在站内音频播放器中播放并切换音质
- 支持在当前搜索结果中切换上一首 / 下一首
- 默认“全部搜索”只检索推荐音乐插件，同时展示完整远程插件目录

当前推荐插件：

- `W音乐`
- `qq`
- `酷我`
- `小秋音乐`
- `小蜗音乐`
- `好听轻音乐`
- `网易`
- `小芸音乐`
- `酷狗`

目录接入说明：

- 页面会同步展示远程 `plugins.json` 里的全部目录条目，但默认只聚合推荐音乐插件做搜索。
- `qq`、`酷我`、`小秋音乐`、`小蜗音乐` 当前已验证搜索与播放链路可用，歌词能力仍可能受到上游插件实现影响。
- `好听轻音乐` 当前已验证支持搜索、播放与推荐榜单，可直接作为推荐音乐插件使用。
- `网易` 当前已验证支持搜索、播放、歌词与推荐榜单，可直接作为推荐音乐插件使用。
- `酷狗` 当前已验证支持搜索与推荐榜单，播放通过搜索匹配接入 `网易` 播放链路。
- `小秋音乐` 当前通过兼容映射接入 `qq` 的播放链路，`小蜗音乐` 当前通过兼容映射接入 `酷我` 的播放链路，`小芸音乐` 当前通过兼容映射接入 `网易` 的播放链路。
- 已确认无法稳定运行的目录插件会在服务层直接移除，不再展示到音乐库页面。
- 电台、听书、歌词站、本地库等非音乐搜索插件也会显示在目录区，但不会参与当前歌曲搜索。

### 设置页 `settings.html`

设置页用于维护片源配置。

主要能力：

- 添加片源
- 编辑片源
- 删除片源
- 切换当前片源
- 测试单个片源接口
- 批量执行健康检查

## 核心模块

核心逻辑集中在 `js/app.js`。

### `CONFIG`

统一维护基础配置：

- 本地代理地址 `PROXY_URL`
- 内置默认片源 `DEFAULT_SOURCES`
- 本地存储键名 `STORAGE_KEYS`

### `Storage`

对 `localStorage` 做了简单封装，统一处理：

- 读取 JSON
- 保存 JSON
- 删除数据

### `ThemeManager`

负责亮色 / 暗色主题切换，并同步页面上的主题按钮状态。

### `SourceManager`

负责片源配置管理：

- 读取所有片源
- 自动补齐内置片源
- 获取当前片源
- 按 `id` 获取片源
- 新增 / 编辑 / 删除片源
- 持久化当前片源选择

### `Api`

负责前端请求封装，所有接口请求都先走本地代理：

1. 浏览器请求 `/api/proxy?url=目标接口`
2. 本地服务转发到目标 CMS 接口
3. 再将结果返回前端

同时这里也负责：

- 通用请求参数拼接
- 接口错误解析
- 单片源健康检查
- 重试逻辑控制

### `music-plugin-service.js`

负责 MusicFree 音乐插件接入：

- 拉取并缓存远程 `plugins.json`
- 仅启用白名单插件
- 在 Node 侧加载远程插件代码
- 使用 `curl` 兼容插件内部网络请求
- 统一输出歌曲搜索结果和播放地址

### `DataAdapter`

负责把不同数据源返回的数据适配为统一结构。

当前支持：

- `vod` 影视源
- `art` 资讯源

适配结果会统一补充这些来源字段：

- `source_id`
- `source_name`
- `source_kind`
- `source_url`

### `VideoParser`

负责把 CMS 返回的播放字段解析成前端可渲染的数据结构：

- `vod_play_from`
- `vod_play_url`
- 剧集列表
- 播放源列表

### `VideoMeta`

负责内容展示层的派生信息计算，例如：

- 最近更新
- 连载中
- 已完结
- 高清
- 是否含 M3U8
- 标签文案
- 描述摘要
- 筛选匹配

### `SearchAggregator`

负责执行多片源搜索并汇总结果。

当前行为：

- 并发搜索所有已配置片源
- 记录成功和失败的片源
- 做轻量去重
- 按时间、封面、摘要优先级排序
- 根据结果内容识别返回类型是 `vod`、`art` 还是 `mixed`

### `PosterCache`

通过本地缓存减少重复请求详情接口来获取封面图。

### `UI`

提供通用界面渲染方法，例如：

- 加载态
- 空状态
- 错误态
- Toast 提示
- 卡片渲染
- 分页渲染

## 本地代理服务

`proxy-server.js` 是项目的运行核心，负责同时提供静态资源和接口代理。

### 静态资源服务

- 根路径 `/` 默认返回 `index.html`
- 允许访问项目根目录下的静态文件
- 通过 MIME 类型映射返回正确的响应头
- 通过路径规范化避免目录穿越

### 代理接口

代理接口格式：

```text
/api/proxy?url=https://example.com/api.php/provide/vod/?ac=list
```

主要逻辑：

- 只允许 `GET` 请求
- 只允许代理 `http` / `https` URL
- 使用 `curl` 发起真实请求
- 自动附加浏览器 UA、语言和 Referer
- 支持读取系统代理设置
- 将上游返回内容尽量按 JSON 响应给前端

### 错误映射

代理层会将常见网络错误映射成更友好的错误信息，例如：

- 连接超时
- 网络不可达
- 域名解析失败
- 目标服务器拒绝连接
- 请求超时

这部分信息会回传给前端，用于提示用户检查片源地址或网络环境。

## 数据源说明

当前内置了一组默认片源，保存在 `js/app.js` 的 `CONFIG.DEFAULT_SOURCES` 中。

片源结构示例：

```js
{
  id: 1,
  name: '量子影视',
  url: 'https://cj.lziapi.com/api.php/provide/vod/',
  active: true
}
```

约定说明：

- `vod` 接口用于影视内容
- `art` 接口用于资讯内容
- `url` 最好指向 CMS 接口基础地址
- 系统会对 URL 做规范化处理

所有片源数据默认存储在浏览器本地，不会写入服务端文件。

## 开发建议

### 调试顺序

建议优先按下面顺序排查问题：

1. 确认 `npm start` 后本地服务已启动
2. 在浏览器打开 `http://localhost:3000`

- `js/app.js` 中的数据适配是否仍兼容 `vod` 和 `art`
- 首页、详情页、播放页之间是否正确传递了 `source_id`
- 新增 UI 状态时是否同步处理空态、错误态和加载态

### 修改代理逻辑时

重点关注：

- 是否仍只允许代理合法的 `http` / `https` 地址
- 是否保持静态文件读取安全
- 是否会影响 JSON 自动解析逻辑
- 是否会破坏前端现有的错误处理格式

## 常见问题

### 1. 启动时报 `node: command not found`

说明当前终端没有加载 Node.js 环境。

如果你使用 `nvm`，先执行：

```bash
export NVM_DIR="$HOME/.nvm"
. "$NVM_DIR/nvm.sh"
```

然后再运行：

```bash
npm start
```

### 2. 页面可以打开，但没有数据

通常是以下原因之一：

- 片源地址失效
- 目标站点当前不可访问
- 本机网络或代理设置异常
- 目标接口返回的数据结构不符合当前适配逻辑

建议先去设置页执行“测试连接”或“健康检查”。

### 3. 为什么 `curl -I http://localhost:3000` 返回 405

因为当前服务只支持 `GET` 请求，`HEAD` 请求会返回 `405 Method Not Allowed`。这不影响正常网页访问。

### 4. 为什么详情页或播放页会串到错误片源

排查重点是页面跳转时是否带上了 `source_id`。当前项目依赖这个参数在跨页面时还原正确的数据源上下文。
